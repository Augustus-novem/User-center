# 验证码规则

## 适用范围
本规则适用于登录验证码、注册验证码、找回密码验证码以及其他身份校验验证码场景。验证码可以通过短信、邮件或站内消息等方式发送。

## 基本规则
1. 验证码应绑定具体业务场景，不能在不同场景之间混用。
2. 验证码应设置有效期，超过有效期后自动失效。
3. 验证码校验成功后，应立即失效，避免重复使用。
4. 同一手机号或邮箱在短时间内请求验证码时，应限制发送频率。
5. 验证码连续校验失败达到阈值后，可以触发临时限制。

## 安全规则
1. 验证码不应明文长期保存。
2. 服务端应记录发送时间、使用状态和过期时间。
3. 当请求来源异常时，可以拒绝发送或要求更严格校验。
4. 验证码长度、有效期和频控阈值应由服务配置统一管理。

## 返回说明
1. 发送成功时，返回已发送状态。
2. 校验成功时，返回验证通过状态。
3. 校验失败时，返回验证码错误、已失效或请求过于频繁等提示。
