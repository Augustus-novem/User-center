# 登录规则

## 适用范围
本规则适用于用户中心的账号密码登录、手机号登录以及设备登录校验流程。所有登录行为都应经过基础校验，并记录必要的审计日志。

## 基本规则
1. 用户发起登录时，服务端需要校验账号是否存在。
2. 若使用密码登录，必须校验密码是否正确。
3. 若账号连续多次登录失败，可以触发临时限制策略。
4. 若检测到异常设备或异常地区登录，可以要求进行二次验证。
5. 登录成功后，应更新最近登录时间、登录设备信息和登录 IP。

## 风险控制
1. 当同一账号在短时间内出现多次失败登录时，系统可以限制继续尝试。
2. 当同一设备频繁切换账号登录时，系统可以记录风险行为。
3. 当登录来源明显异常时，系统可以要求验证码校验。

## 返回说明
1. 登录成功时，返回登录成功标识和用户基础信息。
2. 登录失败时，返回明确但不过度暴露的信息，例如账号或密码错误。
3. 命中风控时，返回需要二次验证或稍后再试的提示。
